- **collectors/** — `BaseCollector` ABC + `CollectionResult` dataclass. `SECEdgarCollector` extends `BaseCollector`, implements `collect(target_date) -> CollectionResult`, and delegates SEC feed parsing/storage to `src/collectors/sec_edgar/`. The Atom feed helper now post-filters exact ownership forms (`4` and `4/A`) because SEC's `type=4` current-filings query also returns unrelated `4*` forms such as `424B2`.
- **tools/** — `ToolContext` (holds optional `session` + runtime `config`), `BaseTool` ABC, and `ToolRegistry`. `build_research_tool_registry()` now wires 9 tools: `MarketDataTool`, `NewsDataTool`, `GlobalContextTool`, and 6 database-backed insider query tools. `GlobalContextTool` builds a replayable macro/global-news block: FRED-backed oil/gold/yields/credit-spread/VIX indicators, White House official updates, Trump-related official updates filtered from the same official stream, and AP world-news geopolitical headlines.
- **prompts/** — `Prompt` is a frozen dataclass with `id`, `version`, `template`, and `description`. `PromptRegistry` lazily loads YAML prompt definitions from `src/prompts/templates/`; the bundled prompt is `research_v1.yaml`.
- **agents/** — `BaseAgent` carries the tool registry, prompt registry, and `model_name`. `ResearchAgent` validates `ResearchInputPayload`, renders the YAML prompt, calls a thin Phidata runner that selects Gemini for `gemini*` model IDs and `OpenAIChat` otherwise, coerces JSON, and validates `StructuredResearchOutput`. Runtime `time_horizon` is currently fixed to `1d` for same-day feedback. The default model is `RESEARCH_MODEL_NAME` or `gemini-2.5-flash-lite`, and Google auth resolves from `GOOGLE_API_KEY`.
- **db/** — SQLAlchemy models live under `src/db/models/` and include `InsiderTrade`, `Watchlist`, `ResearchRun`, `ResearchOutput`, and `EvalResult`. Alembic revision `004_research_app_tables.py` creates the research tables and constraints.
- **research/** — `ResearchPipeline` builds replayable `input_json` snapshots and persists runs/outputs. Each scheduled batch now fetches one shared `global_context` snapshot and copies it into every run for replayability; single-ticker manual runs default to reusing the latest same-day `global_context` unless `--refresh-global-context` is passed. `EvalPipeline` performs same-day scoring: pre-open runs use `open_to_close`, post-open manual runs use `run_time_price_to_close` with the ticker entry taken from persisted input JSON.
- **scheduler/** — `BaseJob` + `JobConfig` define APScheduler jobs, and `SchedulerService` registers `SECEdgarJob`, one pre-open `ResearchJob`, and one `EvalJob` on a blocking scheduler.
- **core/** — `src/core/config.py` loads repo-root `.env` configuration and exposes database, SEC EDGAR, scheduler, and research-model settings. `src/core/logging.py` configures stream logging plus optional rotating-file output.
- **scripts/** — `scripts/run_scheduler_service.py` initialises migrations and starts the SEC + research + eval scheduler set. `scripts/run_research_once.py` and `scripts/run_eval_once.py` are one-off batch entrypoints, while `scripts/run_research_agent_once.py` remains a direct no-DB smoke script for `ResearchAgent`.

**Why:** Tool context is passed per invocation instead of living on long-lived tool instances. Tools are registered explicitly, prompt loading is lazy and file-backed, and same-day signal iteration is kept deterministic by storing run-time input snapshots before eval.
